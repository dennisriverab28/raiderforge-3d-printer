## Overview

RaiderForge is a custom firmware for a fully custom-built FDM (Fused Deposition Modeling) 3D printer designed and assembled from scratch. The MCU is an STM32G0B1RET6 (ARM Cortex-M0+). The printer uses CoreXY kinematics and a direct-drive extruder. The firmware handles all real-time motion control, heating, G-code parsing, display output, and SD card reading вЂ” with no off-the-shelf firmware like Marlin involved.

## Firmware Features

- **CoreXY Motion Control** вЂ” coordinated X/Y stepper motion with acceleration ramping
- **Homing Sequence** вЂ” endstop-based axis homing for X, Y, Z
- **G-code Parser** вЂ” reads G-code from SD card and dispatches commands
- **Heater Control** вЂ” PID-based hotend and bed temperature regulation with thermistor feedback
- **Fan Control** вЂ” part cooling and hotend cooling fan PWM control
- **Extruder Control** вЂ” direct drive filament feed/retract with calibration routines
- **Display** вЂ” real-time telemetry (temperatures, position, print status) on Ender 3 screen
- **SD Card** вЂ” FATFS-based SD card read for G-code files (one file at a time)
- **FreeRTOS** вЂ” multitask RTOS architecture for concurrent subsystem management
- **UART Debug Menu** вЂ” interactive serial debug interface for per-subsystem testing

## Build & Flash

### Requirements
- [STM32CubeIDE](https://www.st.com/en/development-tools/stm32cubeide.html)
- ST-Link V2 debugger/programmer
- STM32G0B1RET6 target board

### Steps
1. Open STM32CubeIDE в†’ `File > Open Projects from File System`
2. Select the `RaiderForge_3D_Printer` folder
3. Build: `Project > Build All` (`Ctrl+B`)
4. Flash: connect ST-Link в†’ `Run > Debug`

### Loading a Print File
1. Format a micro SD card as FAT32
2. Copy a single `.gcode` file to the root of the card
3. Insert the card before power-on вЂ” firmware reads it automatically
